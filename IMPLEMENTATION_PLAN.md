## 二、功能扩展设计

### 2.1 扩展目标

将 DeepResearch 从"纯联网研究工具"升级为"联网研究 + 内部资料分析"的完整企业研究助手。

### 2.2 两大核心功能

#### 功能 A：文件上传与分析

用户在对话框上传文件（PDF/Word/Excel/PPT/Markdown/TXT），Agent 读取内容后，结合联网搜索一起生成研究报告。

**用户场景：**
- 上传公司年报 + 联网搜索行业数据 → 生成综合分析报告
- 上传竞品内部资料 + 联网搜索公开信息 → 生成竞品深度分析
- 上传技术文档 + 联网搜索最新进展 → 生成技术趋势报告

#### 功能 B：企业知识库管理

将每次研究的成果、上传的文件、搜索的结果都存到知识库，支持：
- 知识库文档的上传、查看、删除
- 研究过程中自动检索知识库已有信息
- 知识库内容与联网搜索结果的融合

#### 文件解析方案

参考 project_test 的 document_processor.py，使用轻量级解析库：

| 格式 | 库 | 说明 |
|------|-----|------|
| PDF | PyPDF2 | 提取文本，处理扫描件需额外 OCR |
| Word (.docx) | python-docx | 提取段落和表格 |
| Excel (.xlsx) | openpyxl | 提取单元格数据 |

| TXT | 直接读取 | 纯文本 |

分块策略：
- chunk_size: 512 tokens
- chunk_overlap: 64 tokens
- 保留文件名作为来源标记

## 五、数据流设计

### 5.1 文件上传数据流

```
用户上传文件
  → POST /api/upload → 存储文件到本地
  → 触发后台任务：解析→分块→向量化→写入 Milvus
  → 返回文件 ID

用户提交研究课题 + 文件 ID
  → POST /api/research { messages, uploaded_files: [file_id1, file_id2] }
  → task_queue 入队
  → graph 执行：
      generate_plan:
        - 从 Milvus 检索文件相关 chunks
        - 注入到研究上下文
        - 生成研究计划
      research:
        - 结合文件内容 + 联网搜索
        - 生成搜索查询（避免重复文件已有信息）
        - 执行搜索 + 评估
      write:
        - 综合文件内容 + 搜索结果
        - 生成最终报告
```

### 5.2 知识库检索数据流

```
研究过程中：
  generate_queries:
    - 从 Milvus 检索与主题相关的已有 facts
    - 标注"知识库已有"避免重复搜索
    - 生成搜索查询（补充知识库缺少的信息）

  web_search:
    - 搜索完成后，提取新 facts
    - 自动存入 Milvus（已有功能）

  critique:
    - 评估信息充分性时，考虑知识库已有信息
```

## 六、API 设计

### 6.1 文件上传

```
POST /api/upload
Content-Type: multipart/form-data

请求：
  file: 二进制文件
  title: 可选，文档标题

响应：
{
  "id": 1,
  "title": "2025年Q1财报.pdf",
  "file_type": "pdf",
  "process_status": "parsing",
  "created_at": "2025-07-05T18:00:00"
}
```

### 6.2 文件处理状态

```
GET /api/upload/{file_id}/status

响应：
{
  "id": 1,
  "title": "2025年Q1财报.pdf",
  "process_status": "done",
  "chunk_count": 42
}
```

### 6.3 知识库文档列表

```
GET /api/knowledge?page=1&size=20

响应：
{
  "total": 15,
  "items": [
    {
      "id": 1,
      "title": "2025年Q1财报.pdf",
      "source_type": "upload",
      "file_type": "pdf",
      "chunk_count": 42,
      "created_at": "2025-07-05T18:00:00"
    }
  ]
}
```

### 6.4 删除知识库文档

```
DELETE /api/knowledge/{id}

响应：
{ "message": "已删除" }
```

## 九、风险与注意事项

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 大文件解析超时 | 上传 100+ 页 PDF 可能很慢 | 后台异步处理，前端轮询状态 |
| 文件格式兼容性 | 某些 PDF/Word 可能解析失败 | 优雅降级，报错提示用户 |
| Milvus 向量维度 | 需要与现有 facts 保持一致 | 使用相同的 embedding 模型 |
| 知识库数据量增长 | 大量文档导致检索变慢 | 限制单用户文档数量（200个），定期清理 |
| 文件安全 | 用户上传恶意文件 | 限制文件类型和大小（<50MB） |

## 十二、验收标准

### 功能验收

1. 用户可以在 InputForm 中上传 PDF/Word/Excel/PPT/Markdown/TXT 文件
2. 上传后文件被异步解析，状态实时更新
3. 带文件提交研究课题时，Agent 读取文件内容并融入研究
4. 最终报告同时引用文件内容和联网搜索结果
5. 知识库管理页面可以查看、搜索、删除已上传的文件
6. 研究过程中自动检索知识库已有信息，避免重复搜索
7. 研究结果自动提取 facts 存入知识库，供后续研究复用

### 非功能验收

1. 文件上传支持 50MB 以内的文件
2. 文件解析异步执行，不阻塞前端
3. 单用户文件数量限制合理（200 个）
4. 知识库检索响应时间 < 2 秒
5. 所有验证清单项目通过
